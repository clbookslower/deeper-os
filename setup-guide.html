1부 · 먼저, 무엇을 만드는지 이해하기

우리가 만드는 것 — 내 노트를 아는 세컨브레인, '사막여우'

이번에 우리는 내 노트를 직접 읽고 쓰는 세컨브레인을 만듭니다(그 세컨브레인을 함께 길들이는 여우가 '사막여우'예요). 보통 챗봇과 무엇이 다른지부터 보면 쉬워요.

  • 보통 챗봇 — 매번 새 대화. 나를 기억 못 해, "나는 누구고 뭘 한다"를 늘 다시 설명해야 해요.
  • 사막여우 — 내 노트 폴더를 직접 봐요. 그래서 나를 기억하고, 내가 쌓은 글을 읽고, 새 노트를 만들어 저장해요. 복붙이 없어요.

🦊 사막여우는 우리 모두의 공통 동반자예요. 디퍼 문법 커리큘럼을 함께 걷는 여우 한 마리 — 이름은 다 같이 '사막여우'로 키워요. (문법을 함께한 수료의 표식 같은 거예요.)

도구는 세 가지

도구한 줄 정의맡는 일
옵시디언무료 노트 앱내 생각이 텍스트 파일로 쌓이는 곳
클로드 데스크탑 앱 (Code 탭)내 폴더에서 일하는 AI사막여우가 작동하는 자리
디퍼 OS그 AI에 방법론을 더하는 플러그인사막여우의 인격·매일 루틴·30일 여정을 설치

세 가지는 옵시디언(노트가 쌓이는 바닥) → 클로드 Code 탭(노트를 다루는 AI) → 디퍼 OS(AI에게 일하는 법) 순서로 얹혀요. 설치 순서도 이대로예요.

미리 알아두면 쉬운 말 (개념 미리보기)

  • 볼트(Vault) — 옵시디언에서 내 노트들이 담긴 폴더 하나. 앞으로 "볼트"=내 노트 폴더.
  • 입력창 vs 터미널 — 의미·안녕 같은 건 채팅 입력창에, git·gh 같은 영어 명령은 터미널에. (대부분 입력창만 써요.)
  • 플러그인 / 스킬 — 기능 묶음(=디퍼 OS) / 그 안의 방법론(의미·목소리·지형도·여정).
  • 씨앗 — 혼자 읽어도 이해되는 주장 한 문장. 디퍼 OS의 기본 단위. 매일 쌓여 자산이 돼요.
  • Git / GitHub — 파일 변화를 기록하는 도구 / 그 기록을 인터넷 비공개 금고에 백업하는 곳. 내가 직접 안 써요 — 사막여우가 대신.

오늘의 약속

💡 어려운 명령어는 사막여우가 대신 하고, 나는 내용 보고 승인(Accept)만 해요. 무슨 일이 일어나는지는 이 가이드로 함께 이해하고요. 그리고 사막여우는 내 말 그대로 남기고, 무엇을 남길지는 늘 내가 골라요.

🦊 사막여우는 이 볼트에서만 살아요. 나중에 클로드로 다른 일을 할 땐 다른 폴더를 열면 돼요 — 폴더가 다르면 안 섞여요(열 때마다 곧장 사막여우예요). 이 볼트는 나만의 것이라 회사·공유 드라이브에 두지 않고, 비공개로 백업해요(6단계). → 열면 바로 사막여우 · 나만의 볼트 · 비공개 백업, 셋 다.

2부 · 설치하기 (순서대로 따라가기)

위에서부터 한 단계씩. 막히면 그 단계 이름으로 [4부]를 펴보세요.

0단계 · 준비물 (5분)

  • 클로드 유료 구독(Pro 이상) — Code 탭은 유료에서 열려요. claude.ai에서 플랜 확인.
  • 옵시디언 — 무료 노트 앱(1단계에서 설치). obsidian.md
  • PC — 🍎 맥(macOS 13 이상) 또는 🪟 윈도우(Windows 10 이상). 아이패드·폰은 안 돼요. 인터넷 연결.

0.5단계 · Git 설치 (5분)

Git = 파일 변화를 기록·백업하게 해주는 작은 도구. 직접 쓸 일은 없고 사막여우가 뒤에서 써요. 한 번만 깔면 됩니다.
꼭 먼저 깔아두세요 — 나중에 깃허브 백업을 인증할 때(일회용 번호 받기, 6단계) 이게 깔려 있어야 돼요.

  • 🪟 윈도우(꼭) — git-scm.com/downloads/win → 받은 파일 더블클릭 → 화면이 여러 장 이어져도 전부 기본값 그대로 Next → Install → Finish. (설치하면 'Git Bash'도 같이 깔려요 — 같은 거예요.)
  • 🍎 맥(확인만) — 보통 이미 있어요. 없으면 4부 '폴더를 선택해도 작동이 안 돼요(Git 문제)'에서 사막여우가 알려줘요(그때 "설치" 한 번).

1단계 · 옵시디언 설치하고 '볼트' 만들기 (7분)

옵시디언이 처음이셔도 괜찮아요. 여기서 차근차근 같이 해요.

옵시디언이 뭐예요?

글·생각을 적는 무료 노트 앱이에요. 특별한 점 하나 — 모든 노트가 내 컴퓨터 안의 '폴더 + 텍스트 파일'로 저장돼요. 남의 서버에 갇히지 않아요. 그래서 나중에 사막여우가 그 파일을 직접 읽고 써요. (메모장과 비슷한데, 노트를 예쁘게 모아 관리해주는 도구라고 보시면 돼요.)

📌 볼트(Vault) = 내 노트가 담기는 폴더 하나. 옵시디언은 '볼트' 단위로 노트를 모아요. 우리는 디퍼OS용 볼트 하나를 만들 거예요.
이미 옵시디언을 쓰는 분 → 쓰던 볼트(또는 새 폴더) 하나면 돼요. 2단계로 가셔도 됩니다.

① 설치

  1. 브라우저에서 obsidian.md → 가운데 Download 클릭.
  2. 🍎 맥: 받은 .dmg를 열고 옵시디언 아이콘을 응용 프로그램(Applications)으로 끌어다 놓기.

🪟 윈도우: 받은 설치 파일을 더블클릭 → '다음'만 누르면 끝.

  1. 옵시디언을 실행해요.

② 볼트(보관소) 만들기

  1. 처음 켜면 나오는 화면에서 Create new vault(새 보관소 만들기) 클릭. (화면이 영어로 떠도 당황 마세요.)
  2. Vault name에 이름을 적어요 — 예) 나의 디퍼OS.
  3. Browse(위치 선택)를 눌러 저장 위치를 골라요 → 나중에 찾기 쉽게 '문서(Documents)' 폴더 안을 추천해요.
  4. Create → 끝! 텅 빈 볼트가 열려요.

③ 화면 둘러보기 (1분만)

  • 왼쪽 = 내 노트 목록(파일 탐색기). 지금은 비어 있어요.
  • 가운데 = 노트를 쓰는 공간.
  • 새 노트 만들기 = 왼쪽 위 새 노트 아이콘, 또는 Ctrl(맥은 Cmd)+N. 제목 적고 본문을 쓰면 돼요.
  • 저장은 자동이에요 — 따로 누를 필요 없어요. 노트 하나가 작은 텍스트 파일(.md) 하나예요.
  • 한 번 만들어보세요. 틀려도 괜찮아요 — 익숙해지는 게 목적이에요.

④ 핵심 — 옵시디언과 사막여우는 '같은 폴더'를 봐요

이 볼트 폴더를 곧 3단계에서 사막여우(Code 탭)에게도 가리켜 줄 거예요. 그러면 둘이 같은 폴더를 함께 보게 돼요 — 내가 옵시디언에서 쓴 글을 사막여우가 읽고, 사막여우가 만든 노트를 내가 옵시디언에서 봐요. 따로 옮기거나 복사할 필요가 없어요. (긴 글은 옵시디언에서, 사막여우와의 대화는 Code 탭에서 — 같은 노트로 이어져요.)

🔒 기존 노트가 많은 분 — 세팅은 없는 것만 더하지만, 안심하시도록 시작 전 볼트 폴더를 통째로 한 번 복사해 두세요.

2단계 · 클로드 데스크탑 앱 + Code 탭 (5분)

Code 탭 = 사막여우가 작동하는 자리예요(코딩하는 곳 아님 — 내 노트를 다루는 AI를 켜는 곳).

  1. claude.ai/download → 설치 → 로그인(Pro 이상).
  2. 위쪽 탭에서 Code 누르기.

💡 품질 팁 — 모델은 Opus 4.8 이상으로. 사막여우의 답·의미 선물 문장의 깊이는 모델 성능이 좌우해요. Code 탭 모델 설정에서 Opus 4.8 이상으로 두세요 — 같은 질문에도 훨씬 깊은 한 줄이 나와요. (요금제에 Opus가 없으면, 있는 것 중 제일 좋은 모델로.)

3단계 · 내 볼트 연결하기 (3분)

사막여우가 내 노트를 보려면 "이 폴더에서 일해"라고 가리켜줘야 해요(=연결).

  1. Code 탭 환경을 Local로.
  2. Select folder → 1단계에서 옵시디언으로 만든 바로 그 폴더 선택.
  3. 잘 됐는지 입력창에: 여기 어떤 폴더야? 안에 뭐가 있어? → 내 노트를 읽어주면 OK.

🔗 이게 '옵시디언 ↔ 클로드 코드 연결'이에요. 1단계에서 옵시디언으로 만든 그 폴더를 여기서 고르면, 둘이 같은 폴더를 함께 봐요 — 따로 연결 작업이 없어요(볼트가 옵시디언에서 태어났으니 옵시디언은 이미 이어져 있어요).
💡 혹시 볼트를 옵시디언이 아닌 방법으로 만들었다면(예: 폴더만 따로 생성), 옵시디언에서 한 번만 열어주세요: 옵시디언 좌측 끝 볼트 아이콘(또는 좌하단 볼트 이름) 클릭 → '다른 보관소 열기' → '폴더를 보관소로 열기(Open folder as vault)' → 그 폴더 선택.

💡 Code 탭에서 시작해요(본진). Cowork도 같은 폴더를 읽고 로컬에 저장하지만, 설치·세팅은 Code 탭이 가장 매끄러워요.
💡 사막여우는 만들거나 실행하기 전에 "이렇게 할까요?" 물어요 → 내용 보고 Accept.

4단계 · 디퍼 OS 설치 (앱 안 터미널로)

천천히, 한 줄씩. 우리가 칠 건 몇 줄뿐이에요.

🖥️ '터미널'이 처음이세요? (1분만 읽어요)
터미널은 글로 명령을 시키는 칸이에요. 무섭지 않아요 — 한 줄 치고 Enter가 전부예요.
- 🪟 윈도우는 PowerShell(PS C:\…> 로 시작) · 🍎 맥은 터미널(…% 로 시작). 색은 테마 따라 흰색일 수도, 어두울 수도 있어요.
- 붙여넣기(이 가이드의 명령을 복사해 붙일 때): 🪟 마우스 오른쪽 클릭 또는 Ctrl+V · 🍎 Cmd+V.
- 비밀번호처럼 글자가 안 보여도 정상이에요. 그냥 치고 Enter.
- 틀려도 괜찮아요. 다시 치면 돼요.

4-1. 터미널 열기 (앱 안에서 — 두 가지 방법, 아무거나)

  • 방법 ① 버튼 — Code 탭 화면 오른쪽 위 >_ 아이콘(터미널 모양)을 눌러요.
  • 방법 ② 단축키 — Ctrl + ` (백틱: 키보드 맨 왼쪽 위, Esc 아래·숫자 1 왼쪽 키 — 물결 ~과 같은 키예요). 🍎 맥도 똑같이 Ctrl+`. (다시 누르면 닫혀요.)
  • → 둘 다 같은 터미널 칸이 열려요(화면 아래쪽 또는 오른쪽). 위에 "터미널"이라고 적혀 있어요. (🪟 윈도우는 PowerShell PS C:\…> · 🍎 맥은 zsh …% — 색은 테마 따라.)
  • ⚠️ 이건 윈도우 PowerShell·맥 '터미널' 같은 별도 프로그램이 아니라, 클로드 앱 안에 있는 터미널이에요(맥·윈도우 똑같아요).

4-2. 터미널을 '내 볼트' 폴더로 맞추기 — cd 한 번만 배우기

터미널 맨 앞에 지금 폴더가 보여요: PS C:\...> (🍎 맥은 …%).

  • 그 경로가 내 볼트면 → 그대로 4-3으로.
  • 아니면 cd로 옮겨요. cd는 "이 폴더로 가자"는 뜻이고, 딱 둘만 알면 돼요:
  • cd .. — 한 칸 위로 나가기. (.. 점 두 개 = "한 단계 위 폴더")

예: C:\Users\나\볼트\00-Daily 에서 cd .. → C:\Users\나\볼트

  • cd 폴더이름 — 그 폴더 안으로 들어가기.

예: C:\Users\나 에서 cd 볼트 → C:\Users\나\볼트

  • 한 번에 가기: cd (cd + 스페이스) 친 뒤, 볼트 폴더를 터미널 칸으로 드래그해서 떨어뜨리면 경로가 자동으로 붙어요.
  • 칠 때마다 끝에 Enter. 맨 앞 경로가 내 볼트로 바뀌면 성공이에요.
  • 헷갈리면: cd ..를 한두 번 쳐서 위로 나간 뒤, 보이는 폴더로 cd 폴더이름 하며 한 칸씩 들어가요.

4-3. 클로드 실행 (없으면 한 번만 설치)

방금 연 터미널 칸에는 claude 딱 한 단어만 쳐요(아직 슬래시 명령 아니에요).

  • claude → Enter. 설정 화면이 뜨면 4-4로.
  • "claude 인식 안 됨 / not recognized"가 뜨면 한 번만 설치: npm install -g @anthropic-ai/claude-code → Enter → 끝나면 다시 claude → Enter.

4-4. 첫 실행 화면 넘기기 (거의 Enter, 한 번 로그인)

  1. 테마(색) → 고르고 Enter.
  2. 로그인(유료 인증) → "Log in" → 브라우저 열림 → 유료(Pro) 계정 로그인 → "Authorize/허용" → 터미널로 돌아오기.
  3. 깃허브 연결 화면이 나오면 → 지금은 Skip/나중에 (백업은 6단계).
  4. 보안 안내 → Enter.
  5. "이 폴더를 신뢰?" → Yes/Enter.

→ 화면이 바뀌고 아래에 > 입력 표시가 뜨면, 이제 여기가 '클로드 안'이에요(터미널이 클로드로 바뀐 거예요). 🦊

4-5. 디퍼 OS 설치 (세 줄, 한 줄씩 Enter)

📍 여기가 제일 헷갈리는 곳! 칠 자리를 구분해요 — 세 칸이 달라요.
① Code 탭 채팅창 — 여기선 /plugin이 안 돼요("isn't available in this environment"). ❌
② 터미널 칸(4-1에서 연 것) — 여기엔 claude만 쳤어요.
③ 그 터미널 안에서 실행된 '클로드'(> 입력 표시) — 아래 /plugin 명령은 꼭 ③ 여기에 쳐요. ✅
👉 한 줄 요약: 터미널 열기(버튼/Ctrl+`) → claude 실행 → 실행된 클로드 안에 /plugin…

③ 클로드 안(> 표시 보이죠?)에서 한 줄씩 치고 Enter:

/plugin marketplace add https://github.com/clbookslower/deeper-os.git
/plugin install deeper-os@deeper-os
/reload-plugins
  • 중간에 "추가/설치할까요?" 물으면 Yes/Enter.

4-6. 확인

  • /deeper-os:setup 쳐서 반응하면 성공 → 5단계로.

🔄 이미 디퍼 OS를 쓰던 분(또는 1기) — 다시 설치 말고, 클로드 안에서 /plugin marketplace update deeper-os → /reload-plugins로 업데이트만. 기존 노트는 그대로예요.

5단계 · 사막여우 깨우기

/deeper-os:setup
  • 맨 먼저 "제가 뭐라고 불러드릴까요?" 하고 호칭을 물어요 — 그 호칭으로 계속 불러줘요(그냥 '디퍼님'도, 이름도, 좋아하는 호칭도 OK). 이어서 직업·역할 정도를 대화하듯 따뜻하게(딱딱한 양식 아님). 사막여우 이름은 자동.
  • 기존 노트는 안 건드리고, 매 단계 "만들까요?" 물어요 → 네.
  • 폴더(00-Daily·01-Grammar 등)·데일리 템플릿·질문이 생기고 —
  • 끝나는 순간 사막여우가 다정하게 인사하며 첫 질문을 건네요. 🦊 (그게 시작이에요. 한 줄 답하면 첫 씨앗이 쌓여요.)

6단계 · GitHub 백업 연결 (7분)

OneDrive 대신 GitHub 비공개 금고가 백업이자 '되돌리기'예요. 어려운 명령은 사막여우가 대신 하고, 나는 딱 둘만:

  1. (한 번) GitHub 계정 — github.com → Sign up → 가입(무료) → 이메일 인증.
  2. (한 번) 연결 맡기기 — (Git이 깔려 있어야 돼요 → 0.5단계.) 입력창에 백업 연결 → 사막여우가 도구 설치·저장소 생성·첫 백업까지 해줘요. 진행 중 내가 할 일:
  • 🔑 일회용 번호(코드)로 GitHub 인증 — 터미널(또는 사막여우)이 번호(코드)를 보여주고 브라우저를 열어요. → 그 번호를 복사 → 브라우저에 붙여넣기(🪟 우클릭 또는 Ctrl+V · 🍎 Cmd+V) → GitHub 로그인 → Authorize(승인) → 터미널로 돌아오면 자동으로 이어져요. 평생 한 번.
  • 🍎 (맥) 중간에 비밀번호 창이 뜨면 맥 로그인 비번 입력(글자 안 보이는 게 정상) + Enter.
  1. (이후 매일) 백업 한마디 → 사막여우가 비공개 금고에 저장.

⚠️ 백업은 건너뛰지 마세요. OneDrive를 안 쓰니 이게 유일한 안전장치예요. 저장소가 비공개(Private)인지 한 번만 확인하면 끝 — 나만 봐요.

✅ 여기까지 오면 설치 끝! 이제 3부로.

4부 · 막힐 때 (문제 해결)

디퍼 OS가 안 깔려요 (/plugin이 "isn't available in this environment")

Code 탭 채팅에서 /plugin이 막히는 환경이 있어요. 그럼 Code 탭 안의 통합 터미널에서 claude를 거쳐 설치하면 확실해요. (윈도우 PowerShell이나 맥 '터미널' 앱 같은 별도 프로그램 말고, 클로드 앱 안에 있는 터미널이에요 — 맥·윈도우 똑같아요.)

  1. 통합 터미널 열기 — Ctrl + 백틱 키(키보드 숫자 1 왼쪽, 작은따옴표 비슷하게 생긴 키)를 누르면 아래에 까만 칸이 열려요. 🍎 맥도 같은 키예요 — Cmd가 아니라 Ctrl + 백틱. 이 터미널은 지금 연 볼트 폴더에서 바로 열려요 — cd 안 해도 돼요.
  2. claude 입력. ("claude 못 찾음"이면 앱/클로드 코드 업데이트.)
  3. 맨 처음 한 번은 '첫 설정'이 떠요(다음부턴 안 떠요). 거의 Enter로 통과:
  • 테마(색) → Enter / 로그인=유료계정 확인(구독 계정 로그인) / 보안 안내 → Enter / Git 확인 / "이 폴더 신뢰?" → Enter
  1. 그 안에서 4단계 세 줄 → /deeper-os:setup.

💡 더 쉬운 자동 길 — 볼트에 .claude/settings.json 파일을 만들고 아래를 넣으면, claude 열 때 신뢰 한 번으로 자동 설치돼요(아래 박스만 그대로 복사):

{ "extraKnownMarketplaces": { "deeper-os": { "source": { "source": "git", "url": "https://github.com/clbookslower/deeper-os.git" } } },
  "enabledPlugins": { "deeper-os@deeper-os": true } }

업데이트했는데 옛 버전 그대로예요

  • 채팅 /plugin이 막히면 → 위처럼 통합 터미널 claude에서 marketplace update → install → /reload-plugins.
  • 그래도면 ＋ → Plugins에서 deeper-os 삭제 후 재설치. (노트·볼트 CLAUDE.md는 그대로 — 플러그인만 다시 깔려요.)
  • 내 버전 확인 — 클로드 안에서 /plugin 목록의 deeper-os 버전이 최신인지 보거나, 사막여우에게 "지금 디퍼 OS 버전 알려줘". (세팅을 다시 안 돌려도 돼요 — '이름 묻는지'로 구별하던 옛 방법은 이미 설치한 분껜 안 맞아요.)
  • 🔄 1기(이미 쓰던 분) 한 가지 더 — 업데이트 뒤 사막여우에게 "내 CLAUDE.md 명령어 설명을 지금 플러그인 기준으로 맞춰줘" 한마디. 옛 명령 뜻(특히 오아시스)만 현행으로 바꾸고 내 소개·씨앗·노트는 절대 안 건드려요.

세팅이 "볼트 폴더 경로를 알려달라"고 물어봐요

claude가 볼트가 아닌 곳(예: C:\ 최상위)에서 열린 거예요. 경로 외울 필요 없어요 — Code 탭에서 '내 볼트 폴더'를 다시 열고(Select folder) /deeper-os:setup을 다시 부르면 안 물어요. (터미널이면 cd로 위로 가지 말고 볼트에서 그대로 claude.)

폴더를 선택해도 작동이 안 돼요 (Git 문제)

  • 🪟 윈도우 — Git for Windows가 깔려 있어야 해요(0.5단계). 설치 후 앱 다시 켜기.
  • 🍎 맥 — git이 없을 수 있어요. 통합 터미널에서 claude를 한 번 띄우면, 없을 때 "명령어 도구(Command Line Tools) 설치" 창이 떠요 → 설치 누르고(몇 분) 끝나면 다시 시도.

gh auth login을 직접 입력하게 될 때 (선택지)

위에서부터 차례로: GitHub.com → HTTPS → Yes → Login with a web browser → 일회용 코드 복사·Enter → 브라우저에서 코드 붙여넣고 Authorize.

자주 묻는 것

  • Code 탭이 "업그레이드하라" → 유료 구독(Pro 이상) 필요.
  • /deeper-os:setup을 못 알아들어요 → /reload-plugins 한 번 더.
  • 기존 노트가 사라질까 봐 → 세팅은 없는 것만 추가, 기존 파일 안 건드려요. 불안하면 볼트 폴더 복사.
  • 매일 직접 노트를 만들어야? → 아니요, 안녕 한마디면 사막여우가.
  • GitHub에 올리면 남이 보나요? → 아니요, 비공개(private) — 나만 봐요.
  • (옵시디언) 사막여우가 만든 노트가 안 보여요 → 보통 자동으로 떠요. 안 보이면 왼쪽 목록에서 그 노트를 다시 클릭하거나, 옵시디언을 닫았다 다시 열어보세요(같은 폴더라 곧 나타나요).
  • (옵시디언) 영어 메뉴라 어려워요 → 거의 안 건드려요. 새 노트 만들기와 적기만 하면 돼요. 나머지 정리·저장은 사막여우가 해줘요.
  • (옵시디언) 노트는 어디에 써요? → 짧은 답은 채팅(사막여우)에, 긴 글은 옵시디언에서. 같은 노트로 이어져요(3부 💬 참고).

부록

Code · Cowork · Chat 같이 쓰기

  • Code(본진) — 내 볼트를 읽고 쓰고, 나를 기억함. 남길 작업 전부 여기.
  • Chat — 파일 없이 빠른 대화. 좋은 답 나오면 Code로 가져오기.
  • Cowork — 오래 걸리는 일(리서치 등)을 맡기는 도우미. 폴더를 읽고 로컬에 저장도 돼요. 길게 돌릴 일은 여기서.

탭끼리는 기억을 공유 안 해요 → 좋은 결과는 복사해 Code 사막여우에게 붙여넣고 "정리해줘". 최종 자산은 늘 Code에 남겨요.

용어 한 줄 사전

  • 옵시디언 / 볼트 — 노트를 텍스트로 쌓는 앱 / 그 노트 폴더 하나.
  • Code 탭 / Local / Cowork — 사막여우가 작동하는 자리 / 내 컴퓨터(로컬) / 오래 일하는 도우미(폴더 읽고 로컬 저장).
  • 사막여우 — 나를 기억하고 노트를 다루는 나의 세컨브레인(우리 모두의 공통 동반자, 여우).
  • 플러그인 / 스킬 / /명령 — 기능 묶음(디퍼 OS) / 방법론 / 내가 부르는 기능.
  • 씨앗 / 의미의 축 — 주장 한 문장 / 모든 작업의 기준이 되는 한 축.
  • Git / GitHub / 커밋 — 변화 기록 도구 / 인터넷 비공개 금고 / 저장 시점 묶기.
