# 1인 기업 창작자를 위한 세컨 브레인 — deeper OS

디퍼 언브랜딩 살롱의 사유 자산화 시스템을, **나만의 세컨브레인(사막여우)** 형태로 묶은 저장소입니다.
멤버는 클로드 데스크탑 앱 안에서 명령 세 줄로 스킬·여정·세팅 커맨드를 설치합니다.
OneDrive·팀 공유 없이 **혼자 쓰는 개인 볼트**를 전제로 하며, 백업은 사막여우가 GitHub 비공개 저장소로 대신합니다.

## 설치 (멤버용)

1. **클로드 데스크탑 앱**을 받아 로그인하고(Pro 이상), **`Code` 탭**을 엽니다.
   - 데스크탑 앱에는 클로드 코드가 내장되어 있어 터미널·Node를 따로 설치하지 않아도 됩니다.
   - 윈도우는 [Git for Windows](https://git-scm.com/downloads/win)가 설치돼 있어야 로컬 폴더에서 작동합니다.
2. `Local` 환경에서 **내 옵시디언 볼트 폴더**를 선택합니다.
3. Code 탭 입력창에 **한 줄씩** 차례로 입력합니다.

```
/plugin marketplace add https://github.com/clbookslower/deeper-os.git
/plugin install deeper-os@deeper-os
/reload-plugins
```

> 설치가 목록에 안 보이면 입력창 옆 `＋` → `Plugins`에서 `deeper-os`를 찾아 설치하세요.

## 세팅 전에 — 기존 노트는 안전합니다

`/deeper-os:setup`은 **기존 노트·폴더를 절대 수정하거나 지우지 않습니다.** 없는 폴더·파일만 새로 추가하고, 매 단계 확인을 받습니다. 그래도 한 달 넘게 쌓아온 볼트라면, 세팅 전에 **볼트 폴더를 통째로 한 번 복사**해 백업해 두시길 권합니다. (옵시디언 볼트는 그냥 폴더라서, 복사 한 번이면 완전한 백업입니다.)

## 초기 세팅

```
/deeper-os:setup
```

대화로 CLAUDE.md(사막여우 소개·호칭·업무)·폴더 구조·데일리 템플릿·질문 40개를 만들어 줍니다.
이어서 30일 문법 여정과 GitHub 백업도 연결할 수 있습니다. (의미의 축은 30일 여정 끝에 세워요.)

## 매일 쓰는 법

- `오늘 데일리노트 만들어줘` — 사막여우가 00-Daily에 오늘 노트를 만들고 단계에 맞는 질문을 제안
- `디퍼 여정 시작` → 매일 `오늘 여정 가자` — 30일 문법 여정을 하루씩 진행
- `내 의미의 축 찾고 싶어` / `내 문체 분석해줘` / `내 지식 지형도 그려줘` / `내 독자랑 오퍼 정리해줘` — 스킬 자동 호출
- `백업해줘` — 사막여우가 GitHub 비공개 저장소로 백업(커밋·업로드)

## 구성

```
deeper-os/
├── .claude-plugin/
│   ├── marketplace.json     마켓플레이스 정의 (name: deeper-os)
│   └── plugin.json          플러그인 정의 (name: deeper-os)
├── skills/                  공통 방법론 — 필요할 때 자동 로드
│   ├── meaning/             의미   · 로고테라피 의미 발굴
│   ├── voice/               목소리 · 언어 지문 5층위 분석
│   ├── topography/          지형도 · 지식 지형도 + 콘텐츠 추천
│   ├── branding/            브랜딩 · 이상적 독자·IP 확장·오퍼
│   └── journey/             여정   · 문법 30일(4주) 데일리 여정 엔진
├── agents/
│   └── writing.md           서브에이전트 예시 — 각자 매체로 복제
├── commands/
│   └── setup.md             /deeper-os:setup — 초기 세팅(사막여우·폴더·여정·백업)
└── README.md
```

## 사용 흐름

- **첫 달(문법)은 `journey`(30일 여정)를 따라가며** 나를 모으고(나 탐구) 비웁니다(오토파지). 매일 `오늘`·`여정`.
- **의미의 축은 그 끝(Day30)에 `meaning`으로** 세우고, 둘째 달부터 `topography`·`branding`으로 구조를 짓습니다.
- 서브에이전트는 자기 매체대로 이름 붙여 만듭니다. (예: 글쓰기·유튜브·뉴스레터)
- 모든 분석·추천은 `01-Grammar/Insights/_의미의축.md` 를 기준점으로 삼습니다.

## 갱신

저장소를 수정해 푸시하면, 멤버는 Code 탭에서 마켓플레이스를 새로고침하여 최신 버전을 받습니다.
