Deeper Unbranding Salon

1인 기업 창작자를 위한

deeper OS
설치 · 연결 · 운영 가이드 — 2026

흩어진 생각을 파일로 꺼내 쌓고, 그것을 다시 콘텐츠와 상품으로 길어 올리는 시스템입니다. 옵시디언이 저장소가 되고, 클로드 코드가 그 안에서 읽고 씁니다.

이 가이드는 설치부터 첫 발행까지 전 과정을 안내합니다. 한 달간 사용한 볼트와 폴더 구조는 그대로 이어 쓰며, deeper OS는 플러그인 하나로 설치됩니다.

Why It Matters

왜 이것이 자산이 되는가

머릿속 생각은 붙잡지 않으면 사라집니다. deeper OS는 그 생각을 파일로 꺼내(외재화) 한곳에 쌓고, 서로 잇습니다. 꺼내 둔 생각은 흩어지지 않고, 새 생각과 만나 가치가 불어납니다. 시간이 복리로 일합니다.

차이가 하나 더 있습니다. AI 채팅에만 기대면, 대화가 끝나는 순간 쌓은 것도 함께 사라집니다. deeper OS는 모든 것을 내 볼트에 남기므로, 모델이 바뀌어도 자산은 내 것으로 남습니다. AI에 종속되지 않고, 내 맥락을 소유하는 구조입니다.

이것을 컨텍스트 엔지니어링이라 부릅니다. AI에게 매번 나를 설명하는 대신, 내 맥락 — 원천·씨앗·의미의 축·규칙 — 을 한 번 구축해 두면, AI는 늘 내 맥락 위에서 일합니다. 좋은 질문을 던지는 것(프롬프트)을 넘어, AI가 일할 환경을 설계하는 일입니다.

그래서이런 맥락 자산을 쌓는 창작자는, 시간이 갈수록 누구도 복제할 수 없는 자기만의 깊이를 갖습니다. AI 시대의 진짜 차별화는 여기서 나옵니다 — 도메인 지식 × AI 역량.
Two Connection Paths

두 가지 연결 방식

옵시디언에 Claude를 연결하는 길은 둘입니다. deeper OS는 두 번째를 표준으로 삼습니다.

① 옵시디언 플러그인 + Anthropic API 키

옵시디언 안에 Copilot, Smart Connections 같은 플러그인을 설치하고 API 키를 입력하는 방식입니다. 화면 안에서 가볍게 질문하기 좋지만, 스킬과 에이전트를 쓸 수 없고 사용한 토큰만큼 과금됩니다. 가벼운 질문용 보조로 두셔도 됩니다.

② 클로드 코드 + obsidian-mcp (표준)

Claude Pro 또는 Max 구독으로 로그인하므로 별도 API 키나 토큰 과금이 없습니다. 스킬·서브에이전트·볼트 읽기·쓰기를 모두 지원합니다. deeper OS의 본 작업이 이 방식에서 구현됩니다.

Storage

저장 구조 — 작업은 로컬, 공유는 OneDrive

옵시디언과 OneDrive를 함께 쓰면 동기화 충돌이 생기기 쉽습니다. 평소 작업은 PC 로컬에서 하고, 공유하기로 정한 인사이트만 OneDrive로 보냅니다. 폴더는 트리비움 세 단계 — 발굴(문법) · 구조(논리) · 확산(수사) — 를 그대로 따릅니다.

deeper-salon/ ← 로컬 작업 공간 ├── 01-Grammar/ 문법 · 씨앗을 발굴한다 │ ├── Raw-Text/ 원천 (원본 그대로 보관) │ └── Insights/ 씨앗 (자기완결적 주장 + 키워드 링크) ├── 02-Logic/ 논리 · 씨앗을 비즈니스 구조로 │ ├── Reader/ 이상적 독자와 고객 여정 │ ├── Offer/ 상품·오퍼·가격 │ ├── Map/ IP 확장 지도 (글→책→강의→커뮤니티) │ └── Routine/ 지속 가능한 창작 루틴 ├── 03-Rhetoric/ 수사 · 씨앗을 세상으로 │ ├── Channel/ 채널 실험과 선택 │ ├── Published/ 발행한 글 아카이브 │ └── Manifesto/ 매니페스토와 다음 분기 방향 ├── CLAUDE.md 지침 (설치 시 자동 생성) └── .claude/ 스킬·에이전트 (플러그인이 설치)
↓ 발행 결정한 씨앗만 클로드 코드가 복사
OneDrive / 디퍼살롱1기-공유아카이브/ ├── 그룹인사이트/ 그룹과 함께 보는 인사이트 └── 디퍼유레카_각자의 발견/ 각자의 발견 기록
왜 이렇게 나누나01에서 씨앗을 모으고, 02에서 그 씨앗을 묶어 누구에게(Reader)·무엇을(Offer)·어디까지(Map)·어떻게 지속(Routine)할지 비즈니스 구조를 세우고, 03에서 채널로 내보냅니다. 지속 가능한 1인 브랜딩에 필요한 골격이 폴더로 자리 잡습니다.

기존에 OneDrive 안에 볼트를 두셨다면, 로컬로 옮기는 작업은 설치 후 클로드 코드에 요청할 수 있습니다. 폴더는 /deeper-os:setup이 만들어 줍니다.

Install

설치 — 네 단계

명령어는 Windows 기준입니다. 맥은 해당 단계의 별도 명령어를 사용합니다.

Step 1
클로드 코드 설치 및 로그인

데스크톱 앱의 Code 탭에서 터미널을 열고 입력합니다.

# Windows (PowerShell)
irm https://claude.ai/install.ps1 | iex

# 맥 (Terminal)
curl -fsSL https://claude.ai/install.sh | bash

claude를 입력해 실행하고, 브라우저가 열리면 Claude 계정으로 로그인합니다.

확인시작 화면이 표시됩니다. 점검은 claude doctor.

Step 2
Node.js 설치

nodejs.org에서 LTS 버전을 기본 설정으로 설치하고 확인합니다.

node -v
확인v20 이상이면 정상입니다.

Step 3
옵시디언 볼트 연결

로컬 볼트 경로를 확인한 뒤(예: C:\Users\사용자\Documents\deeper-salon), claude 터미널에 요청합니다.

내 옵시디언 볼트를 obsidian-mcp 서버로 연결해 줘.
경로는 C:\Users\사용자\Documents\deeper-salon 야.
모든 작업에서 쓰게 설정하고, 확인 방법을 알려 줘.
확인/mcp 입력 시 obsidian이 connected로 표시됩니다.

Step 4
deeper OS 설치 — 깃헙 주소로

deeper OS는 github.com/clbookslower/deeper-os 저장소에 올라가 있습니다. 아래 주소를 클로드 코드 대화창에 넣어 마켓플레이스로 등록하고 플러그인을 설치합니다. 한 줄씩 차례로 입력합니다.

/plugin marketplace add https://github.com/clbookslower/deeper-os.git
/plugin install deeper-os@deeper-os
/reload-plugins

첫 줄이 저장소를 마켓플레이스로 등록하고, 둘째 줄이 deeper OS를 설치하며, 셋째 줄이 활성화합니다. 짧은 clbookslower/deeper-os 형식은 일부 환경에서 SSH로 연결을 시도해 실패하므로, 위의 https:// 전체 주소를 사용합니다.

안 보이면설치가 목록에 안 뜨면 /plugin 메뉴 → Marketplaces → deeper-os → Install for you (user scope) 로 설치합니다.

이어서 초기 세팅을 실행합니다. CLAUDE.md·폴더 구조·의미의 축을 대화로 만들어 줍니다.

세팅 전에 — 기존 노트는 안전합니다/deeper-os:setup은 기존 노트·폴더를 수정하거나 지우지 않습니다. 없는 폴더·파일만 새로 추가하고, 매 단계마다 확인을 받습니다. 그래도 한 달 넘게 쌓아온 볼트라면, 마음 편히 시작하시도록 세팅 전에 볼트 폴더를 통째로 한 번 복사해 백업해 두시길 권합니다. 이미 자기 구조로 정리하셨다면 폴더 생성은 건너뛰어도 됩니다.
/deeper-os:setup
확인CLAUDE.md와 폴더가 생기고, 설치된 스킬 목록이 안내됩니다.
What's Inside

구성과 에이전트 설정

deeper OS는 네 층으로 작동합니다. 공통 방법은 살롱이 제공하고, 개인화는 각자 설계합니다.

CLAUDE.md — 항상 켜진 맥락

소개·작성 규칙·메인 에이전트 이름·의미의 축 참조. 작업할 때마다 클로드 코드가 먼저 읽습니다.

스킬 — 공통 방법론

의미·목소리·지형도·브랜딩. 대화 맥락에 맞게 자동으로 불려 옵니다.

서브에이전트 — 개인화된 작업자

글쓰기·유튜브·뉴스레터처럼 자기 매체대로 만듭니다. 스킬을 활용합니다.

MCP — 볼트 연결

obsidian-mcp가 클로드 코드와 옵시디언을 잇습니다. (Step 3)

스킬 4종

스킬단계하는 일
의미문법로고테라피로 ‘의미의 축’을 길어 올린다
목소리문법글을 5층위로 분석해 언어 지문과 희석 신호를 가린다
지형도문법→논리지식 지형도를 그리고 콘텐츠를 추천한다
브랜딩논리이상적 독자·변환·IP 확장·오퍼를 설계한다

에이전트 설정 — 메인과 서브

메인 에이전트 — 이름을 정합니다

메인 에이전트는 클로드 코드 그 자체입니다. 볼트를 총괄하고, 스킬을 부르고, 서브에이전트에 일을 맡깁니다. /deeper-os:setup이 CLAUDE.md에 이름을 적어 둡니다. 부르고 싶은 이름을 정하면 됩니다.

서브에이전트 — 매체별로 하나씩, 세 단계로

1단계 · 만들기를 요청합니다. 클로드 코드 대화창에 이렇게 말합니다.

글쓰기를 맡을 서브에이전트를 만들고 싶어. /agents로 만들어 줘.

2단계 · 이름과 설명을 정합니다. 이름(키)은 영문 소문자, 설명은 한글로 적습니다.

이름(키):  writing
설명:      블로그·에세이 초안을 내 목소리로 쓴다

3단계 · 무엇을 읽고 쓸지 알려 줍니다.

내 씨앗(01-Grammar/Insights)과 의미의 축을 읽고,
voice·branding 스킬을 활용해. 발행 글은 '나'·'종합' 씨앗만 써.

유튜브를 한다면 youtube, 뉴스레터라면 newsletter를 같은 방식으로 만듭니다. 매체마다 하나씩 두면, 부를 때 그 작업자가 스킬을 불러 일합니다.

How to Use

스킬과 에이전트, 어떻게 쓰는가

deeper OS는 클로드 코드 안에서 작동합니다. 데스크톱 앱 Code 탭에서 클로드 코드를 실행하고, 볼트에 연결된 그 대화창에서 사용합니다. claude.ai 웹 대화창이 아닙니다.

스킬 — 한글로 말하면 됩니다

스킬은 대화 맥락에 맞게 자동으로 불려 옵니다. 슬래시나 영문을 외울 필요가 없습니다.

"내 원천 읽고 목소리 분석해 줘"   →  목소리 (voice)
"지형도 그려 줘"                  →  지형도 (topography)
"의미를 찾고 싶어"                →  의미 (meaning)
"이걸로 뭘 만들까"                →  브랜딩 (branding)

특정 스킬을 콕 집고 싶으면 이름을 직접 부르면 됩니다 — "voice 스킬로 분석해 줘"처럼. 슬래시 명령으로 등록되는 것은 /deeper-os:setup 같은 커맨드뿐이고, 스킬은 대화로 부릅니다.

이렇게 흘러갑니다

나: Raw-Text에 어제 메모 넣었어. 읽고 씨앗으로 정리해 줘.
  → 원천을 읽고 자기완결적 주장(씨앗)으로 Insights에 저장, 키워드 링크.

나: 지형도 그려 줘.
  → 씨앗 문장·원천·씨앗을 읽고 지식 지형도 + 콘텐츠 추천.

나: 추천한 첫 글, writing 에이전트로 초안 잡아 줘.
  → writing 에이전트가 목소리·브랜딩 스킬을 참고해 초안 작성.
The Unit

씨앗 — deeper OS의 기본 단위

deeper OS의 기본 단위는 문서가 아니라 ‘씨앗’, 즉 하나의 주장 문장입니다. 각 씨앗은 단독으로 읽어도 이해되도록 씁니다. 그리고 출처를 표기합니다 — 나 · 인용 · 종합. 외부 발행 글은 ‘나’와 ‘종합’ 씨앗으로만 쓰고, ‘인용’은 자료로만 둡니다. 본인의 문장과 인용이 섞이지 않게 하여 목소리를 일관되게 지키는 원칙입니다.

옵시디언의 그래프는 노트 사이의 내부 링크로 그려집니다. 씨앗마다 핵심 키워드를 [[키워드]]로 표시하면, 같은 키워드를 단 씨앗들이 그 키워드 노드로 모여 주제별 색인이 됩니다. 클로드 코드는 CLAUDE.md 규칙에 따라 저장할 때마다 키워드를 자동으로 링크합니다. 작업이 쌓일수록 지도가 스스로 자라납니다.

Publish

발행 — 공유 아카이브로 내보내기

작업은 로컬에서 하고, 공유하기로 정한 씨앗만 OneDrive 공유 아카이브로 보냅니다. 클로드 코드에 파일과 폴더를 지정해 요청합니다.

01-Grammar/Insights의 [파일명] 씨앗을 공유 아카이브의
'그룹인사이트' 폴더로 복사해 줘. 로컬 원본은 그대로 둬.

로컬 원본은 보존되고, 복사본만 동기화되어 그룹이 봅니다. 기본은 비공개이며, 발행은 명시적으로 요청할 때만 일어납니다.

생각을 꺼내 쌓으면 복리로 불어난다. AI에 기대지 않고, 내 맥락을 소유한다.

Troubleshooting

문제가 생겼을 때

오류가 나면 claude doctor로 설치 상태를 점검합니다. /mcp에 obsidian이 없으면 클로드 코드를 종료했다 다시 실행합니다. 플러그인이 보이지 않으면 /reload-plugins를 실행합니다. 그래도 막히면 그 화면을 저장해 살롱에서 함께 점검합니다.
