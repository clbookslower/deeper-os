사용자의 볼트에 디퍼 OS를 세팅한다. 각 단계에서 사용자에게 확인받는다.

## 데이터 안전 원칙 (반드시 먼저 지킨다)

- 기존 노트·폴더·파일을 **절대 수정·삭제·이동·덮어쓰기 하지 않는다.** 오직 '없는 것'만 새로 추가한다.
- 모든 생성 단계는 만들기 전에 무엇을 만들지 사용자에게 보여주고 확인받는다. "아니오"면 건너뛴다.
- 파일·폴더가 이미 있으면 손대지 않는다. 내용이 다르면 덮어쓰지 말고, 무엇을 더하면 좋을지 알려만 준다.
- 이미 자기만의 구조로 정리한 사용자는 폴더 생성을 통째로 건너뛰어도 된다고 먼저 안내한다.

0. 시작 전, 사용자에게 안내한다:
   "이 세팅은 기존 노트를 건드리지 않고 없는 폴더·파일만 새로 추가합니다. 그래도 마음 편히
    시작하시도록, 볼트 폴더를 통째로 한 번 복사해 백업해 두시길 권합니다. 진행할까요?"
   사용자가 동의할 때만 다음 단계로 넘어간다.

1. 볼트 최상위에 CLAUDE.md가 **없을 때만** 아래 내용으로 만든다. [ ]는 사용자에게 묻는다.
   이미 CLAUDE.md가 있으면 **절대 덮어쓰지 않는다.** 빠진 항목이 있으면 "추가하면 좋을 내용"으로
   보여만 주고, 합칠지는 사용자가 직접 정하게 한다.

```
[소개]
- 직업·역할: (사용자 입력)
- 한두 줄 소개: (사용자 입력)
- 현재 디퍼 언브랜딩살롱에서 나만의 원천 텍스트를 발굴하는 중입니다.

[메인 에이전트]
- 이름: (사용자 입력)
- 역할: 볼트 전체를 총괄하고, 작업에 맞는 스킬을 부르고, 서브에이전트에 일을 위임한다.

[씨앗 — 작성 규칙]
1. 원천(원고·기억·인용 원본)은 01-Grammar/Raw-Text에 원본 그대로 보관한다.
2. 씨앗 하나는 단독으로 읽어도 이해되는 한 문장으로 01-Grammar/Insights에 저장한다.
   (씨앗 = 디퍼 OS의 기본 단위. 자기완결적 주장 한 문장.)
3. 각 씨앗에는 출처를 표기한다: 나 / 인용 / 종합.
4. 외부 발행 글은 '나'와 '종합' 씨앗으로만 작성하고, '인용'은 자료로만 쓴다.
5. 씨앗마다 핵심 키워드(개념·인물·주제)를 [[키워드]] 내부 링크로 표시한다.
   한 씨앗에 3~5개 이내. 인용 인물·출처도 [[이름]]으로 링크한다.
6. 발행 요청 시, 지정한 씨앗을 OneDrive 공유 아카이브의 해당 폴더
   (그룹인사이트 또는 디퍼유레카_각자의 발견)로 복사한다. 로컬 원본은 보존한다.

[의미의 축]
- 01-Grammar/Insights/_의미의축.md 를 모든 작업의 기준점으로 삼는다.
- 스킬과 서브에이전트는 분석과 추천을 이 축에 비추어 우선순위를 정한다.
- 의미의 축은 'meaning' 스킬로 갱신한다.

[문체]
정확하고 명료하게. 군더더기와 상투구를 덜어낸다.
```

2. 다음 폴더 중 **없는 것만** 만든다. 만들기 전에 목록을 보여주고 확인받는다.
   (이미 다른 구조로 정리한 사용자는 건너뛰어도 된다.)
   - 01-Grammar/Raw-Text      (원천)
   - 01-Grammar/Insights       (씨앗)
   - 02-Logic/Reader           (이상적 독자·고객 여정)
   - 02-Logic/Offer            (상품·오퍼·가격)
   - 02-Logic/Map              (IP 확장 지도)
   - 02-Logic/Routine          (지속 가능한 창작 루틴)
   - 03-Rhetoric/Channel       (채널 실험·선택)
   - 03-Rhetoric/Published     (발행 글 아카이브)
   - 03-Rhetoric/Manifesto     (매니페스토·다음 분기 방향)

3. 'meaning' 스킬을 안내하여, 사용자가 원하면 지금 의미의 축을 발굴하고 _의미의축.md 로 저장한다.

4. 사용자에게 자기 매체에 맞는 서브에이전트를 만들지 묻는다. 원하면 agents/writing.md 를
   본보기로, name(영문 키)과 description(한글)을 바꿔 새 서브에이전트를 만든다.

세팅이 끝나면, 설치된 스킬 목록과 다음에 할 일을 짧게 안내한다.
