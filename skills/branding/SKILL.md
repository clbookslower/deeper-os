# 브랜딩 — 브랜드 확장 설계

씨앗 문장·원천·씨앗을 읽고 순서대로 설계한다. 상품은 정보가 아니라 변환(transformation)이다.

1. 이상적 독자 — 이 목소리가 가장 깊이 닿는 한 사람의 구체적 초상.
2. 변환 — 그 독자의 '이전 상태 → 이후 상태'. 무엇이 달라지는가.
3. IP 확장 지도 — 씨앗 하나가 글 → 책 → 강의 → 커뮤니티로 뻗는 경로를 6개월 / 1년 / 3년 단위로 가지친다.
4. 오퍼 초안 — 지금 당장 만들 수 있는 가장 작은 상품 하나. 무엇을, 누구에게, 왜.

항상 씨앗 문장과 01-Grammar/Insights/_의미의축.md 에서 출발한다. 수익 구조보다 의미의 일관성을 먼저 본다.
설계가 끝나면, 이상적 독자·변환은 `02-Logic/Reader`, IP 확장 지도는 `02-Logic/Map`, 오퍼 초안은 `02-Logic/Offer` 에 저장할지 사용자에게 묻는다. (없는 폴더만 새로 만들고, 기존 파일은 덮어쓰지 않는다.)
